# ADR-0006: 歌声 F0 推定に YIN 法を採用する

## Status

Accepted (2026-07-16) — 実装済み (フェーズ3:
`src/acoustics/core/` VocalAnalyzer)。

## Context

- フェーズ3 の歌声分析は F0 時系列を基礎とする (ピッチ安定性・
  ビブラート・HNR・倍音抽出がすべて F0 に依存)。オペラ歌唱は
  F0 レンジが広く (Bass 低音 ~70 Hz から Soprano 高音 ~1400 Hz)、
  強いビブラート・豊富な倍音 (歌手フォルマント) を持つため、
  オクターブ誤りに強い推定器が必要。
- 検討した選択肢:
  - **素朴な自己相関法 (ACF)**: 実装最小だが、倍音の強い歌声では
    2 倍周期 (1 オクターブ下) のピークを拾うオクターブ誤りが頻発する。
  - **YIN** (de Cheveigné & Kawahara 2002): 自己相関と数学的に等価な
    差分関数を用いつつ、**累積平均正規化 (CMNDF)** により τ = 0 付近
    のバイアスと倍音由来の偽ピークを抑える。「閾値を下回る最初の
    極小」という絶対閾値則がオクターブ誤りを系統的に減らす。論文の
    式だけから自前実装でき、追加依存ゼロ (C++14 コアに収まる)。
  - **pYIN**: YIN の確率的拡張 (HMM 平滑化)。精度は上がるが実装量が
    大きく、Viterbi 平滑化はオフライン指標 (中央値・RMS 統計) には
    過剰。
  - **ケプストラム法**: フレーム内で FFT×2 が必要な割に低 F0 の分解能
    が悪く、高 F0 (ソプラノ) でケフレンシーピークが不明瞭。
  - **CREPE 等の学習ベース**: 精度最高クラスだがニューラルネット
    推論系 (モデル配布・依存) が必要で、依存ゼロ方針 (ADR-0001 /
    ADR-0005) に反する。
- ライセンス: YIN はアルゴリズム (論文) からの自前実装であり、
  第三者コードを含まない (`docs/licensing-review.md` §3 で確認済み)。

## Decision

1. **YIN 法を採用する** (差分関数 → CMNDF → 絶対閾値による最初の
   極小探索)。定義は `docs/opera-acoustic-metrics.md` §10.2。
2. **パラメータ** (VocalAnalyzer の既定値):
   - フレーム長 40 ms / ホップ 10 ms (Bass 下限 70 Hz でも 1 フレーム
     に約 3 周期入る長さ)。
   - CMNDF 閾値 0.15 (論文の推奨域 0.1–0.15 の上限側 — ビブラートに
     よる周期揺らぎで有声フレームを落としすぎない)。
   - **無声判定**: `min d'(τ) > 0.15`、またはフレーム RMS が RMS
     ゲート未満。無声フレームは F0 を出力しない (ダミー値で埋めない)。
3. **声種 (`voice_type`) は探索範囲の限定のみに使う**:
   Sop 220–1400 / Mez 180–1100 / Alt 160–900 / Ten 110–700 /
   Bar 90–500 / Bass 70–400 / Unknown 60–1500 Hz。範囲限定は
   オクターブ誤りをさらに抑えるための**事前情報**であり、既定は
   Unknown (全範囲) で成立する。
4. **診断的結論の禁止**: 本ソフトは声種の自動判定を行わず、発声の
   良否・声の健康状態 (病的嗄声等) に関する結論・助言を表示しない。
   出力は物理量 (F0 統計・cent 偏差・HNR dB 等) と品質 3 値
   (Valid/Warning/Invalid) に限る。HNR 等は声質と相関するが、医学的
   スクリーニングの妥当性検証をしていない値を診断風に見せることは
   誤用リスクが大きい (仮定 §20)。

## Consequences

- (+) 依存ゼロ・C++14 のまま歌声分析コアが完結する。テストも
  合成信号 (既知 F0 + ビブラート変調) で決定的に検証できる。
- (+) CMNDF + 絶対閾値により、素朴な ACF 比でオクターブ誤りが大幅に
  減る (論文 Table III: ACF 系比で誤り率約 1/3 以下)。
- (+) 声種範囲はプリセットにすぎないため、範囲外の歌唱でも Unknown で
  分析できる (機能が声種入力に依存しない)。
- (−) pYIN のような時系列平滑化はないため、単発フレームの外れ値は
  残る。後段指標が中央値・RMS ベースであることで実害を抑えている。
- (−) 重唱・伴奏付き録音では単一 F0 の前提が崩れる (仮定 §18)。
  多重 F0 推定はスコープ外。
- (−) 探索範囲を声種で絞ると、範囲外の実音 (ベースのファルセット等)
  が無声扱いになる。UI は「範囲外が多い場合は Unknown を選ぶ」旨を
  案内する。
- 将来 pYIN 的平滑化や声区分析・フォルマント推定 (F1/F2, LPC) を
  追加する場合も、本 ADR の「診断的結論の禁止」は維持する
  (`docs/opera-acoustics-development-status.md` 負債 #14)。
