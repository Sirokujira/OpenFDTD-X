# ADR-0005: 可聴化は自前 radix-2 FFT + Overlap-Add で実装する (外部 FFT ライブラリ不採用)

## Status

Accepted (2026-07-16) — 実装済み (フェーズ4:
`src/acoustics/core/` の Fft / ConvolutionEngine)。

## Context

- フェーズ4 の可聴化は「ドライ歌唱 WAV × RIR」の線形畳み込みを必要と
  する。RIR はホール規模で数秒 (fs 48 kHz で 10⁵ サンプル級)、直接
  畳み込み O(N·M) は非現実的で、FFT ベースのブロック畳み込みが必要。
- FFT の入手手段として外部ライブラリを検討した:
  - **FFTW3**: 最速級だが **GPL** — 本プロジェクトのライセンス方針
    (`docs/licensing-review.md`) と相容れない (商用配布・ライセンス
    非汚染を維持したい)。
  - **KissFFT (BSD) / PFFFT (BSD 系) / muFFT** など: ライセンスは許容
    範囲だが、C++14 コア (`ofdx_acoustic_core`) の**依存ゼロ**という
    性質 (ADR-0001 — 単独 `g++ -std=c++14` でビルド可、外部カーネル・
    組込みへの持ち出し) を初めて破ることになる。第三者コードの取り込み
    は出所追跡・脆弱性追随・ビルド系統合の継続コストも伴う。
- 必要な FFT の要件は限定的である: 実数系列の 2^n 長 forward/inverse、
  オフライン処理 (リアルタイム制約なし)、LTAS (フェーズ3 Welch 法
  4096 点) と畳み込みブロックの 2 用途のみ。SIMD 最適化された
  ライブラリとの速度差が体感に影響しない規模。
- 畳み込み方式の選択肢: 単発の巨大 FFT (メモリピーク大・進捗報告
  不可)、Overlap-Add / Overlap-Save (ブロック処理・実装素直)、
  Partitioned Convolution (低レイテンシ・リアルタイム向けだが複雑)。

## Decision

1. **FFT は自前実装する** (`Fft`、C++14・依存ゼロ・radix-2 反復型、
   入力長は 2^n に限定)。外部 FFT ライブラリは採用しない。理由は
   ライセンス方針 (GPL 回避) と、コアの依存ゼロ維持 (ADR-0001) を
   優先するため。radix-2 限定で十分: 使用箇所 (LTAS 4096 点・
   畳み込みブロック) はすべてこちらが長さを選べる。
2. **畳み込みは Overlap-Add** (`ConvolutionEngine`)。RIR をブロック長
   に対しゼロ詰めし、ドライ信号をブロック分割して FFT 乗算・重畳加算
   する。ブロック長は RIR 長から 2^n で自動選択。
3. **fs 不一致は明示エラー**: ドライと RIR のサンプリング周波数が
   異なる場合、暗黙のリサンプリングはせずエラーを返す (仮定 §21)。
4. **自動正規化しない**: 出力振幅はスケールせず、`outputPeak` /
   `suggestedGainDb` (ピークを −1 dBFS 等に収める推奨値) /
   クリップサンプル数を結果として報告する。ゲイン適用はユーザー選択
   (`.ofdx` `auralization.gain_mode`)。WAV I/O の無正規化方針
   (仮定 §12) と一貫させる。
5. **A/B 比較はファイル書き出し**: ドライ/ウェット WAV を書き出して
   外部プレイヤーで比較する。リアルタイム再生・切替は初期版では
   実装しない。

## Consequences

- (+) コアの依存ゼロ・C++14 単独ビルド・ライセンス非汚染が維持される
  (GPL コード不使用、第三者コードの取り込みなし)。
- (+) Overlap-Add はブロック単位の進捗報告・キャンセルが自然にでき、
  メモリピークもブロック長で抑えられる。
- (+) 正規化しないため、校正済みワークフロー (Absolute) でも振幅の
  意味が保たれる。クリップリスクは報告値で事前に判断できる。
- (−) FFTW 等と比べ数倍遅い。オフライン用途では許容 (数秒級の RIR ×
  数分の歌唱で対話的速度を確認済みであること — 検証文書に記録)。
- (−) radix-2 限定のため任意長 FFT (素因数分解型) が必要になった場合
  は拡張が要る。現用途では発生しない。
- (−) リアルタイム A/B (即時切替試聴) は未対応。必要になった時点で
  **Partitioned Convolution** (ブロックを不均等分割し先頭を短くする
  低レイテンシ方式) + 音声出力デバイス統合を別フェーズとして実装する
  (残課題 — `docs/opera-acoustics-development-status.md` 負債 #11)。
- 高品質リサンプラーは未実装のまま (負債 #12)。fs 混在素材は外部
  ツールでの変換を案内する。
