# 設計・実装で置いた仮定

本書は実装 (`src/acoustics/**`) と要求 (`docs/opera-acoustics-requirements.md`)
に埋め込まれている仮定を明文化する。仮定が崩れる条件と影響も併記する。
番号は他文書からの参照用。

## 1. ST 系指標は初期版では参考値のみ / 算出不可

ISO 3382-1 の ST_early / ST_late は「舞台上・音源から 1 m の全指向
マイク・空席」という測定条件を要求するが、ユーザー提供の WAV から
この条件をソフトウェアが検証する手段はない。
**仮定**: 初期版 (フェーズ2) では ST 系を計算せず、UI 上は
「参考値 (自己申告時) / 測定条件不適合 / 算出不可」の 3 値表示枠のみを
定義する (要求 §3.2)。本実装はフェーズ4。
**影響**: 舞台支援の定量値は当面、時間区分反射集計 (0–20/20–80 ms) で
代替する。

## 2. G (Strength) は校正なしでは相対値

G は基準音源 (無響室 10 m 自由音場) との比較を要する絶対指標。
**仮定**: `CalibrationState = Absolute` (dBFS→dB SPL オフセット既知) の
場合のみ絶対値を表示し、それ以外は絶対 SPL 系を `valid = false` にする
(実装済み: `RirAnalyzer` の `absoluteSplDb`)。校正オフセットは周波数に
依存しない単一値と仮定する (帯域別校正はスコープ外)。
**影響**: 未校正データでは「声の届きの絶対量」は評価できず、比指標
(C50/C80/D50/Ts/RT) のみで評価する。

## 3. RIR は単一チャンネルとして分析する

**仮定**: 分析対象は常に 1 チャンネル。多チャンネル WAV は
`channel_mode` (0=L / 1=R / 2=全チャンネル平均モノ) で 1 本化してから
分析する (実装済み: `QtAcousticAdapter::selectChannel`。指定チャンネルが
無い場合は先頭チャンネルにフォールバック)。
**影響**: バイノーラル/アンビソニックの空間指標 (IACC, LF 等) は
スコープ外。平均モノは位相打ち消しで高域が低下し得るため、ステレオ
測定では L/R 個別分析を推奨する。

## 4. クリッピング判定は |x| > 0.999 が 3 サンプル連続

**仮定**: フルスケール近傍 `|x| > clipThreshold (0.999)` が
`clipRunLength (3)` サンプル連続した区間をクリッピングとみなす
(実装済み: `RirAnalyzerConfig`)。単発の 0.999 超え (ディザ・デルタ) は
クリッピング扱いしない。検出時は**エラーではなく警告**
("clipping detected: results may be unreliable") で分析を続行する。
**影響**: 軽度の飽和 (2 サンプル以下) や、録音機側で 0.999 未満に
スケールされたクリップ済み信号は検出できない。

## 5. ノイズフロア = 信号末尾 10% の RMS

**仮定**: RIR の末尾では残響が減衰しきってノイズ支配になっている。
末尾区間 (全長の 10%、最小 256 サンプル) の RMS をノイズフロアとする
(実装済み: `estimateNoiseFloor`。入力が短い場合は全体を使い warning)。
**影響**: 録音を残響尾より早く切った WAV ではノイズフロアを過大評価し、
動的範囲不足 (指標 invalid) 側に倒れる — 安全側の失敗。

## 6. ノイズは定常 (Chu 補正の前提)

**仮定**: 付加ノイズは分析区間で定常であり、後方積分から
`p̂·(N−i)` を減算できる (Chu 1978)。補正は指標計算では既定 on。
**影響**: 非定常ノイズ (空調の断続・通過音) では補正が不正確になる。
その場合も回帰品質 (R² < 0.95 → Warning) で検出される可能性が高い。

## 7. 分析終了点は Lundeby 簡略版

**仮定**: 5 ms 移動平均包絡線がノイズパワー +10 dB を下回る最初の点で
回帰範囲を打ち切れば、ノイズ支配域の混入を防げる (完全な Lundeby
反復法は実装しない)。
**影響**: ノイズフロア近傍の減衰 (ダブルスロープ等) では終了点が
保守的になり、T30 が invalid になりやすい (これも安全側)。

## 8. 直接音は「最初の強い到来」

**仮定**: RIR には直接音が含まれ、それは最大ピークか、最大ピークの
−20 dB 以内かつ 1 ms 以上先行するピークである (実装済み:
`DirectSoundDetector`、先行ピーク採用時は Warning)。GUI 既定は
EnvelopeThreshold 法 (−20 dB 閾値)、コア既定は Peak 法。
**影響**: 直接音が遮蔽され −20 dB より弱いケース (舞台袖の測定等) では
最初の強い反射を直接音と誤認し、時間原点がずれる。警告表示で
ユーザー確認を促す。

## 9. DC 除去は全区間平均の減算

**仮定**: DC オフセットは記録系由来の定数であり、平均値減算 (既定 on)
で十分。ハイパスフィルタは掛けない。
**影響**: 超低域ドリフト (風雑音等) は除去されず、低帯域の指標品質に
影響し得る (帯域フィルタで大半は緩和)。

## 10. 帯域フィルタは Butterworth 4次 (ゼロ位相で実効 8次) で十分

**仮定**: ISO 3382-1 の帯域分析に IEC 61260 クラス準拠フィルタの厳密な
実装は要求せず、双一次変換による 4 次バターワース BPF (ゼロ位相
filtfilt で振幅 2 乗) を同等とみなす。帯域上端がナイキスト以上の帯域は
設計失敗として当該帯域のみ算出不可にする (実装済み)。
**影響**: 規格フィルタとの肩特性差により帯域端のエネルギー配分が
わずかに異なる。検証 (帯域分割後 T30 誤差 ≤ 2.02%) では許容内。

## 11. 音速 343 m/s (20 ℃ 近傍) を固定値とする

**仮定**: 既存エコーグラム (`RoomAcoustics`) と将来の音響 FDTD 見積り
(ADR-0004) は c = 343 m/s 固定。温湿度による音速・空気吸収の変化は
モデル化しない (既存統計モデルの空気吸収は 1 kHz 基準の固定比)。
**影響**: 距離→遅延の換算誤差は常温範囲で ±2% 程度。

## 12. WAV は自動正規化しない

**仮定**: 校正の一貫性のため、読み込んだ振幅を一切スケールしない
(整数 PCM のフルスケール ±1.0 換算のみ。float は範囲外の値もそのまま
保持)。書き出しも Float32 はクランプしない (実装済み)。
**影響**: レベルの低い録音では動的範囲チェックがそのまま働く
(正規化で「見かけの改善」をしない)。

## 13. Ts の積分上限は測定信号の末尾

**仮定**: 重心時間 Ts の分母/分子の積分は理論上 ∞ までだが、実装は
信号末尾まで (残響尾+ノイズを含む)。
**影響**: 録音が短い・ノイズが多いと Ts は系統的に偏る。検証では
2 反射モデルで理論値と一致 (±1 ms)、人工 RIR で RT/13.82 と一致
(±0.02 s)。

## 14. 人工 RIR (指数減衰白色雑音 + デルタ) を精度検証の基準とする

**仮定**: `h(t) = δ(t−t0) + Σ aᵢδ(t−t0−τᵢ) + n(t)e^(−6.91(t−t0)/RT) +
noiseFloor·w(t)` (決定的 LCG、シード固定) が検証に十分な標準モデルで
ある (`tests/acoustic_data/metadata.json` に条件を記録し、WAV は
コミットせず実行時生成)。
**影響**: 実ホールの RIR (ダブルスロープ、モード共振、時変ノイズ) の
網羅ではない。実測データによる追加検証はフェーズ4 以降。

## 15. 校正状態は 3 値・オフセットは単一値 (未永続化)

**仮定**: `calibration_state` (0/1/2) で表示制御には十分。
`calibrationOffsetDb` はコア API には存在するが GUI 設定・.ofdx には
まだ無い (技術的負債 #1、`docs/opera-acoustics-development-status.md`)。
**影響**: 現状 GUI 経由では Absolute を選んでもオフセット 0 dB の
ままになる。フェーズ2 の残作業。

## 16. 反射検出パラメータの既定値

**仮定**: 移動 RMS 窓 0.5 ms / 最小ピーク間隔 2 ms / 直接音比
−40 dB 以上 / ノイズフロア +6 dB 以上突出 / 直接音後 1 ms から検出 /
最大 64 件 (強い順) が、ホール規模の RIR に妥当な既定である
(実装済み: `ReflectionDetectorConfig`)。
**影響**: 小部屋 (反射間隔 < 2 ms) では反射が統合されて数えられる。
パラメータは config で調整可能。

## 17. 分析結果は永続化せず毎回再計算する

**仮定**: 分析は対話操作に十分な速度 (数秒以内) で完了するため、
`.ofdx` には設定のみ保存し結果はキャッシュしない (`Project.h` に明記、
ADR-0003)。
**影響**: WAV 差し替え・設定変更と結果の不整合が構造的に起きない。
巨大 WAV では再分析の待ち時間が発生する (負債 #6 と関連)。

## 18. 歌声分析の入力は単一声・無伴奏 (フェーズ3)

**仮定**: `voice_file` は 1 人の歌唱のみを含む無伴奏 (ア・カペラ)
録音である。YIN (単一 F0 推定) も HNR も LTAS の解釈も、この前提の
上に成り立つ (`docs/opera-acoustic-metrics.md` §10)。
**影響**: 伴奏・重唱・強い暗騒音が混入した録音では F0 の追従誤り、
HNR の過小評価、歌手フォルマント比の汚染が起きる。ソフトウェアは
混入を検出しない (有声率の低下・HNR 低値として間接的に現れるのみ)。
UI は「無伴奏録音を使用すること」を明記する。

## 19. ビブラートは 3–9 Hz の準周期変調 (フェーズ3)

**仮定**: 歌唱ビブラートのレートは 3–9 Hz に収まり、250 ms 移動平均の
デトレンドで音程遷移 (ポルタメント・フレージング) と分離できる
(定義は `docs/opera-acoustic-metrics.md` §10.4)。
**影響**: 3 Hz 未満の極端に遅い揺れ、9 Hz 超のトレモロ様変動は
「ビブラートなし」と報告される。250 ms より速い音程遷移はビブラート
深さに混入し得る。範囲外の検出が必要ならパラメータ拡張が必要。

## 20. voice_type は F0 探索範囲の限定のみに使う (フェーズ3)

**仮定**: 声種設定 (Sop〜Bass/Unknown) の役割は YIN の探索範囲
(オクターブ誤り抑制) に限る。既定 Unknown (60–1500 Hz) でも分析は
成立する。
**影響**: 本ソフトは声種の自動判定・発声の良否・声の健康状態などの
**診断的結論を出さない** (ADR-0006)。実 F0 が設定声種の範囲外の場合、
そのフレームは無声扱いになる — 範囲外が多いときは Unknown を使う。

## 21. 可聴化はオフライン・等 fs・正規化なし (フェーズ4)

**仮定**: A/B 比較は「ドライ WAV とウェット WAV の書き出し」で足り、
リアルタイム再生 (低レイテンシ・パーティション畳み込み) は初期版に
不要。ドライと RIR の fs は一致していることを要求し、不一致は
**明示エラー**とする (暗黙のリサンプリングをしない)。出力は自動
正規化せず、outputPeak / suggestedGainDb / クリップ数の報告のみ行う
(仮定 §12 と同方針、ADR-0005)。
**影響**: fs の異なる素材は外部ツールでリサンプリングしてから使う
必要がある。Float32 出力はクリップしないが、Pcm16 出力ではゲイン
未調整だとクランプされ得る (報告値で事前に判断できる)。

## 22. モックソルバーは出力契約の検証代理 (フェーズ5)

**仮定**: 出力契約 (metadata.json / rir.wav / metrics.json /
solver.log) に準拠した最小のモックソルバー (小型 C 実行ファイル) で
AcousticRunner→契約検証→RirAnalyzer の統合経路を CI テストすれば、
実ソルバー接続時の GUI 側の正しさは保証できる (ADR-0007)。
**影響**: モックが検証するのは**契約と GUI 側の処理**であり、実
ソルバーの物理的正しさ・性能・進捗挙動は対象外。実ソルバーの検証は
ソルバー側リポジトリの責務。契約とモックが乖離すると検証が無意味に
なるため、契約変更時はモックを同時更新すること。

## 23. 音響 FDTD 見積りは λ/10・CFL √3・4 field の概算 (フェーズ5)

**仮定**: `AcousticFdtdEstimator` は c = 343 m/s (§11)、セル寸法
Δx = c/(10·f_max) (λ/10 規則)、3D CFL Δt ≤ Δx/(c·√3)、メモリ =
総セル数 × 4 field (音圧 p + 粒子速度 vx/vy/vz) × double (8 byte) を概算式として十分とする。
**電磁用ステータスバーの見積り式の流用ではなく独立実装**である
(ADR-0004 Decision 5)。
**影響**: 実ソルバーの離散化 (高次スキーム・非一様格子・PML 層) に
よって実所要量は係数倍変わる。見積りは「実行可否の事前警告」用で
あり、正確な予約量ではない。
