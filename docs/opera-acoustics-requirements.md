# オペラ歌手向け室内音響分析 — 要求定義

対象: OpenFDTD-X (branch `claude/opera-acoustics`)。
本書は歌劇場・リサイタルホール・練習室の音響をオペラ歌手の観点で評価する
機能の要求を定義する。実装済み C++14 コア (`src/acoustics/`) の実 API と
整合させており、指標の計算定義は `docs/opera-acoustic-metrics.md`、
アーキテクチャは `docs/opera-acoustics-architecture.md` を参照。

## 1. 背景と利用者

- **利用者**: オペラ歌手・声楽教師・ホール技術者・音響設計者。
- **課題**: 「舞台上で自分の声が聴こえるか (歌いやすさ)」と「客席へ声が
  届くか (投射)」は別の問題であり、既存の RT60 中心の評価では判断できない。
- **方針**: 分析を 3 系統に分け、それぞれ独立した要求とフェーズで扱う。
  - 系統 A: 歌手フィードバック (舞台上の歌いやすさ)
  - 系統 B: 客席投射 (客席での聴こえ方)
  - 系統 C: 歌声信号分析 (歌唱そのものの分析 — フェーズ3)

## 2. 分析データの 3 区分と UI 明示方針 (必須要求)

数値の「出どころ」が異なる結果を混在表示すると誤解を生むため、UI 上で
以下の 3 区分を**常に明示**する (バッジ・ラベル・レポート見出しに区分名を
併記する)。

| 区分 | 出どころ | 既存/新規 | 信頼性の性格 |
|---|---|---|---|
| ① 統計推定 | Sabine/Eyring/Barron 等の統計式 (`src/core/RoomAcoustics`) | 既存 (RoomAcousticsTab) | 設計初期の目安。周波数・位置の粗い平均 |
| ② 実測 RIR | 実測インパルス応答 WAV の分析 (`src/acoustics/` コア) | 新規 (RirAnalysisTab) | 実在の室の実測値。測定品質に依存 |
| ③ シミュレーション RIR | 外部音響ソルバーが生成した RIR (フェーズ5) | 新規 (将来) | モデル精度に依存。実測と同一パイプラインで分析 |

- ②と③は同じ `RirAnalyzer` パイプラインで分析するが、結果表示には
  「実測」「シミュレーション」を必ず区別して表示する。
- ①の値と②③の値を同一の表・同一のプロット系列に無断で混在させない。
  比較表示する場合は区分ラベルを付ける。

## 3. 系統 A: 歌手フィードバック (舞台上の歌いやすさ)

### 3.1 直接音・初期反射の時間区分分析

実装済み `ReflectionDetector` / `ReflectionTimeSummary` の 4 区分
(直接音到来を 0 ms とする) をそのまま採用する:

| 時間区分 | 音響的意味 (歌手視点) | 表示要求 |
|---|---|---|
| 0–20 ms | 直接音と一体化して聴こえる成分。自分の声の支え (自己モニター) | 反射数・合計エネルギー・直接音比 |
| 20–80 ms | 初期反射。声の「返り」。舞台上の歌いやすさ・アンサンブルのしやすさに直結 | 同上 (最重要区分として強調表示) |
| 80–200 ms | 後期反射の立ち上がり。ホール応答の印象 | 同上 |
| 200 ms– | 長遅延反射。レベルが高いとエコー障害 | 同上 + 高レベル時は警告表示 |

- 各反射イベントは到達時刻・直接音比レベル [dB]・エネルギー・信頼度
  (0..1) を持つ (`ReflectionEvent`)。
- 検出できない環境 (ノイズ埋没・動的範囲不足) では区分集計を非表示に
  せず「検出 0 件 (条件: …)」と理由を表示する。

### 3.2 ST 系指標 (Stage Support) の扱い — 表示 3 値規則

ISO 3382-1 の舞台計測指標 ST_early / ST_late は、**音源から 1 m の
全指向マイク・舞台上配置・空席**という厳格な測定条件を要求する。
一般ユーザーの実測 WAV からこの条件をソフトウェアが検証することは
できないため、以下の 3 値表示規則を必須要求とする:

| 状態 | 条件 | 表示 |
|---|---|---|
| **参考値** | ユーザーが「1 m 測定条件で測定した」と明示的に申告した場合 | 値 + 「参考値 (測定条件は自己申告)」バッジ |
| **測定条件不適合** | 申告がない、または申告内容が条件と矛盾する場合 | 値を表示せず「測定条件不適合」と理由 |
| **算出不可** | RIR の長さ・動的範囲・直接音検出が ST の時間窓 (0–10 / 20–100 / 100–1000 ms) を満たさない場合 | 「算出不可」+ コア warning 文字列 |

- 初期版 (フェーズ2) では ST 系は**計算自体を実装せず**、常に
  「算出不可 (未実装)」または要求次第で「参考値」枠のみ設ける
  (仮定の詳細は `docs/opera-acoustics-assumptions.md` §1)。
- ST 系の本実装はフェーズ4 (校正・測定条件ワークフロー) で行う。

### 3.3 その他の歌手フィードバック要求

- 直接音検出結果 (時刻・レベル・品質) を表示する。最大ピークより先行する
  強いピークを直接音として採用した場合は警告文をそのまま表示する
  (実装済み `DirectSoundDetector` の仕様)。
- 初期時間遅れ間隙 (ITDG 相当) は「直接音から最初の検出反射までの遅延」
  として反射リストから導出して表示する。

## 4. 系統 B: 客席投射 (客席での聴こえ方)

### 4.1 指標一覧 (実測/シミュレーション RIR から算出)

すべて実装済み `computeAcousticMetrics()` (ISO 3382-1 準拠、直接音 t=0)
の出力を用いる。計算定義は `docs/opera-acoustic-metrics.md` 参照。

| 指標 | 単位 | 歌手/聴衆にとっての意味 | 実装状態 |
|---|---|---|---|
| EDT | s | 体感残響感 (音楽の響き) | 実装済み |
| T20 / T30 | s | 残響時間 (規格値) | 実装済み |
| C50 | dB | 語音明瞭度 (レチタティーヴォ・台詞) | 実装済み |
| C80 | dB | 音楽明瞭度 (アリア・アンサンブル) | 実装済み |
| D50 | 0..1 | Definition (語音明瞭度の割合表現) | 実装済み |
| Ts | s | 重心時間 (明瞭感/残響感の単一値) | 実装済み |
| Early/Late (50/80 ms) | 線形比 | 早期/後期エネルギー比 (C50/C80 の線形版) | 実装済み |
| G (Strength) | dB | 声の「届き」の絶対量 | フェーズ4 (§4.3 の校正が前提) |

### 4.2 歌手フォルマント帯域 (2–4 kHz) の帯域別分析

オペラ歌手の声が管弦楽を貫通する鍵は 2–4 kHz の歌手フォルマント
(singer's formant) である。帯域セット `BandSet::SingerFormant`
(実装済み) を UI から選択できること:

| 帯域ラベル | 下端–上端 [Hz] | 中心 (幾何平均) |
|---|---|---|
| 2.0-2.5k | 2000–2500 | 2236 Hz |
| 2.5-3.15k | 2500–3150 | 2806 Hz |
| 3.15-4.0k | 3150–4000 | 3550 Hz |
| 2.0-4.0k (統合) | 2000–4000 | 2828 Hz |

加えて既存互換 6 帯域 (125…4k)、1 オクターブ (63…8k)、1/3 オクターブ
(100…5k) を選択できる (`BandSet` 列挙、ADR-0002)。帯域ごとに全指標
(§4.1) と品質区分を表示する。

### 4.3 CalibrationState 3 値による絶対値表示制御

実装済み `CalibrationState` (Absolute / Relative / Uncalibrated) を
UI 表示制御に直結させる:

| 校正状態 | 表示できるもの | 表示してはならないもの |
|---|---|---|
| Absolute (dBFS→dB SPL オフセット既知) | 絶対 SPL、G (フェーズ4)、相対値すべて | — |
| Relative (相対レベルのみ有効) | 相対レベル (直接音比・C50/C80 等の比指標) | 絶対 SPL、G の絶対値 |
| Uncalibrated (未校正) | 比指標・時間指標 (RT/EDT/Ts 等) | 絶対 SPL、G、SPL 目盛りつきプロット |

- コアは Absolute 以外で `absoluteSplDb` を `valid=false` にする
  (実装済み)。UI はこの valid フラグに従い、無効な絶対値を空欄 +
  「未校正のため算出不可」と表示する。
- 自動正規化は行わない (WAV の値をそのまま分析する) — 実装済み方針。

### 4.4 品質 3 値表示 (全指標共通)

コアの `AnalysisQuality` (Valid / Warning / Invalid) を UI に直結する:

- **Valid**: 通常表示。
- **Warning (参考値)**: 値 + 警告アイコン + 理由 (例: 決定係数 R² < 0.95)。
- **Invalid (算出不可)**: 値を出さず理由文字列 (例: 動的範囲不足) を表示。

## 5. 系統 C: 歌声信号分析 (フェーズ3 — 未着手)

歌唱 WAV (`opera_analysis.voice_file`、キーは予約済み) を対象とする。
本フェーズの要求は「予約」であり、実装・詳細設計はフェーズ3 で行う:

| 分析 | 内容 (案) |
|---|---|
| F0 (基本周波数) | 時系列 F0 軌跡。YIN 系アルゴリズムの自前実装 (ライセンス調査済み: `docs/licensing-review.md` §3) |
| ビブラート | F0 軌跡からレート [Hz]・深さ [cent] |
| LTAS | 長時間平均スペクトル。歌手フォルマント帯域 (2–4 kHz) と 0–2 kHz のエネルギー比 (singer's formant ratio) |
| HNR | 調波対雑音比 (声の張り・かすれ) |
| 声種プリセット | `voice_type` (0..5 = Sop/Mez/Alt/Ten/Bar/Bass, 6 = Unknown) による表示帯域・基準値の切替 |

RIR 分析 (系統 A/B) と歌声分析 (系統 C) は独立に実行できること。

## 6. 非機能要求

1. **基準不変**: `docs/opera-acoustics-baseline.md` の基準 (selftest
   1560 checks / 0 failures、.ofd 不可侵、ビルド成功) を壊さない。
2. **コアは C++14 / Qt 非依存** (ADR-0001)。追加の外部ライブラリなし
   (`docs/licensing-review.md` §3)。
3. **永続化は .ofdx の追加キーのみ** (`acoustic.opera_analysis`、
   `docs/opera-acoustics-file-format.md`)。設定のみ保存し、分析結果は
   毎回再計算する (実装済み方針)。
4. **エラーは例外でなくエラーコード** (`AcousticResult<T>` /
   `AcousticErrorCode` 16 値、実装済み)。
5. **国際化**: 新規 UI 文言は I18n キー (日英) を追加する。
6. **決定性**: テストは決定的 (シード固定 LCG の人工 RIR) で、外部
   データファイル・ネットワークに依存しない (実装済み方針)。
